# 需求评审输出模板

> 本模板定义 `/critique` 技能的标准输出格式

---

## 文档头部

```markdown
---
Doc_ID: CRITIQUE-{Sequence}
Title: [需求/想法] 评审报告
Type: /critique
Content: [需求名称/描述]
Perspectives: [评审视角]
Review_Date: [YYYY-MM-DD]
Reviewer: [评审人]
Version: [版本号]
---
```

## 附录

### 评分标准

```markdown
### 评分标准

**1 分**: 非常差，需要重大改进
**2 分**: 较差，有明显问题
**3 分**: 一般，有改进空间
**4 分**: 良好，有小问题
**5 分**: 优秀，无明显问题
```

### 评审方法

```markdown
### 评审方法

**用户视角评审**:
- 用户能理解如何使用吗？
- 流程是否自然？
- 是否有不必要的步骤？
- 错误处理是否友好？

**商业视角评审**:
- 是否解决真实问题？
- 市场需求是否存在？
- ROI 是否合理？
- 是否有竞争优势？

**工程视角评审**:
- 技术上是否可行？
- 是否有技术风险？
- 是否考虑了可维护性？
- 是否考虑了可扩展性？
```
