# 产品路线图质量检查清单

> 本清单用于评估 `/roadmap` 产出的质量

---

## 使用说明

**使用时机**: 完成路线图文档后，发布前自检

**检查方式**: 逐项核对，不符合的项目应修改后再发布

**合格标准**: 核心项 (🔴) 全部通过，建议项 (🟡) 至少 80% 通过

---

## 核心检查项 (🔴 必需)

### 战略对齐

- [ ] **有战略输入**: 路线图基于明确的战略规划
- [ ] **可追溯**: 每个项目能追溯到战略支柱
- [ ] **优先级清晰**: 有明确的 P0/P1/P2 排序
- [ ] **取舍明确**: 清楚说明不做什么

### 时间规划

- [ ] **时间现实**: 时间规划基于真实产能评估
- [ ] **近细远粗**: 近期详细，远期粗略
- [ ] **包含缓冲**: 预留了不确定性缓冲 (≥10%)
- [ ] **考虑依赖**: 关键依赖已识别

### 内容完整

- [ ] **多视图**: 至少包含 Timeline + Portfolio 视图
- [ ] **里程碑明确**: 关键里程碑有清晰定义
- [ ] **成功标准**: 里程碑有可衡量的成功标准
- [ ] **资源分配**: 明确各团队/资源的分配

### 风险管理

- [ ] **假设明确**: 关键假设已明确表述
- [ ] **风险识别**: 主要风险已识别
- [ ] **应对措施**: 每个风险有缓解措施
- [ ] **不确定性说明**: 高不确定性区域有标注

### 质量门控

- [ ] **利益相关者审阅**: 关键利益相关者已审阅
- [ ] **团队认可**: 执行团队认可路线图
- [ ] **更新机制**: 定义了回顾和更新节奏

---

## 建议检查项 (🟡 推荐)

### 完整性

- [ ] **有 TL;DR**: 执行摘要快速理解核心内容
- [ ] **包含维护**: 预留了技术债务处理时间
- [ ] **包含探索**: 预留了创新探索空间
- [ ] **多维度视图**: 按战略、类型、客户等维度展示

### 沟通质量

- [ ] **简洁可读**: 核心内容 20 分钟内可理解
- [ ] **视觉清晰**: 表格、格式清晰易读
- [ ] **语言一致**: 术语使用一致
- [ ] **符号说明**: 状态符号有说明

### 可维护性

- [ ] **版本控制**: 有清晰的版本号
- [ ] **变更历史**: 记录了主要变更
- [ ] **下次更新**: 明确下次更新时间
- [ ] **归档位置**: 明确文档存储位置

## 常见问题检测

### 战略质量检测

| 问题 | 检测方法 | 修改建议 |
|------|----------|----------|
| 无战略对齐 | 项目无法追溯到战略支柱 | 先完成 `/strategic-planning` |
| 优先级混乱 | 所有项目都是"重要" | 使用 RICE 评分重新排序 |
| 缺少取舍 | 没有明确说明不做什么 | 增加"不做什么"部分 |
| 过度承诺 | 没有预留缓冲空间 | 预留至少 10% 缓冲 |

### 可执行性检测

| 问题 | 检测方法 | 修改建议 |
|------|----------|----------|
| 时间不现实 | 未考虑团队产能和依赖 | 基于真实产能重新规划 |
| 无里程碑 | 没有关键检查点 | 定义季度里程碑 |
| 依赖未识别 | 关键依赖未列出 | 补充依赖关系 |
| 缺少负责人 | 重点项目无负责团队 | 补充负责团队 |

### 沟通质量检测

| 问题 | 检测方法 | 修改建议 |
|------|----------|----------|
| 视图单一 | 只有一种视图 | 增加 Portfolio 视图 |
| 过度详细 | 远期也详细到月 | 近细远粗调整 |
| 无变化说明 | 路线图被视为固定承诺 | 强调方向性，标注预期变化 |

## 视图质量检查

### Timeline 视图

- [ ] 时间维度清晰 (季度/月)
- [ ] 重点项目突出显示
- [ ] 优先级一目了然
- [ ] 状态标注完整

### Portfolio 视图

- [ ] 战略支柱完整覆盖
- [ ] 资源分配可见
- [ ] 平衡性可见 (新功能 vs 维护)
- [ ] 年度目标可追踪

### Now-Next-Later 视图

- [ ] 三个层级清晰
- [ ] 相对顺序合理
- [ ] 不承诺具体时间
- [ ] Backlog 有管理
