# 决策支持

<SUBAGENT-STOP>
如果是子代理派发执行此 Skill，直接按下方指令执行，不重新加载入口 Skill。
</SUBAGENT-STOP>

帮助 PM 做出有依据的产品决策——从"我觉得应该选 A"到"基于 X 证据和 Y 约束，A 方案在 Z 条件下更优"。

**核心原则**：好决策不是选对的，是知道为什么选。决策框架的价值在于让选择可追溯、可复盘、可质疑。

## Iron Law（铁律）

| 铁律 | 违反后果 |
| --- | --- |
| 不列出劣势的方案分析无效 | 立即停止——补充每个方案的 ≥ 1 个劣势后继续 |
| 没有假设的决策不可信 | 立即停止——标注所有隐含假设后继续 |
| 只有一个方案的决策不是决策 | 立即停止——生成 ≥ 1 个替代方案后继续 |

## 反合理化

| 你可能在想的 | 真相 |
| --- | --- |
| "用户已经知道要选什么" | 知道选择 ≠ 理解为什么选。决策记录的价值在于"为什么" |
| "这个方案明显更好" | "明显"是偏见。用数据或逻辑证明，不用直觉 |
| "两个方案差不多，随便选" | 差不多 = 决策风险低 ≠ 不需要记录。记录下来，下次不用重新想 |
| "老板说了选 A" | 老板的决策也需要记录依据，否则无法复盘 |
| "时间紧迫，快速选一个" | 快速选可以，但不能跳过记录。10 分钟的决策记录省 10 小时的返工 |

## Entry Mode

### Quick（快速模式）— ≤ 2 个问题后直接产出

**问题 1**: "你要做什么决策？有哪些方案？"
**问题 2**（可选）: "有什么约束或偏好？"

然后：
- 自动生成 ≥ 2 个方案（如果用户只给 1 个，自动生成 1 个替代）
- 对每个方案列出优势/劣势/风险
- 用加权决策矩阵评分
- 给出建议 + 理由
- 标注所有 [假设]

### Expert（专家模式）— 直接产出 + 高级选项

**高级选项**:
- A. 加权决策矩阵 — 多维度加权评分
- B. 假设反转 — "如果关键假设错了，结论会变吗？"
- C. 时间轴分析 — "6 个月后，哪个方案更优？"
- D. 二阶效应 — "选 A 之后，会引发什么新问题？"
- E. 定价决策 — 使用定价策略框架（见 `references/pricing-framework.md`）

**产出增强**:
- 决策矩阵表格
- 假设清单（带置信度）
- 最小可行验证建议
- 不可逆性评估（可逆决策快速做，不可逆决策慎重做）

## 决策框架

根据决策类型自动选择框架：

| 决策类型 | 框架 | 适用场景 |
| --- | --- | --- |
| 方案选择 | **加权决策矩阵** | A vs B vs C，多维度比较 |
| 优先级决策 | **RICE / ICE** | 先做什么后做什么（路由到 pm-rice） |
| 方向决策 | **假设映射** | "如果 X 为真则选 A，如果 Y 为真则选 B" |
| 权衡决策 | **Trade-off 曲线** | 速度 vs 质量、短期 vs 长期 |
| Go/No-Go | **一页纸决策** | 做 vs 不做，列出支持/反对 |
| 定价策略 | **定价框架** | 价值定价 vs 成本加成 vs 竞争定价 + 定价心理学 |

详细框架模板见 `references/decision-frameworks.md`。

## 执行流程

```
触发 pm-decision
    ├── 1. 解析决策问题
    │     ├── 决策类型（方案选择/优先级/方向/权衡/Go-No-Go）
    │     ├── 已有方案列表
    │     └── 约束条件
    ├── 2. 自动选择框架
    ├── 3. 分析方案（优势 + 劣势 + 风险）
    │     └── 标注 [假设] 和置信度
    ├── 4. Iron Law 检查
    │     ├── 每个方案有劣势？
    │     ├── 所有假设已标注？
    │     └── ≥ 2 个方案？
    ├── 5. 产出建议 + 最小可行验证
    └── 6. 交付 + 写入决策记录
```

## 交付前检查

- [ ] ≥ 2 个方案被分析
- [ ] 每个方案有 ≥ 1 个优势和 ≥ 1 个劣势
- [ ] 所有假设已标注 [假设]
- [ ] 关键假设有置信度评估
- [ ] 有明确的推荐 + 理由
- [ ] 有最小可行验证建议
- [ ] 决策记录使用标准格式
- [ ] **推荐方案的执行路径无 Magic Step**
- [ ] **Trade-off 中的技术约束经过验证**（不是假设可行）
