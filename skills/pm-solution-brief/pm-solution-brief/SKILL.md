# 方案概要

<SUBAGENT-STOP>
如果是子代理派发执行此 Skill，直接按下方指令执行，不重新加载入口 Skill。
</SUBAGENT-STOP>

从 PRD/分析文档提炼一页纸方案概要（≤ 500 字）。如果一页纸说不清方案，说明方案本身不够清晰。

**输入**: PRD/分析文档路径
**输出**: 方案概要（一页纸）

## Iron Law

| 铁律 | 违反后果 |
| --- | --- |
| 超过 500 字 = 方案不清 | 删减——如果需要更多篇幅，回到问题理解阶段 |
| 没说"不做什么" = 假装能做所有事 | 立即补充——明确权衡和排除项 |
| 无量化指标 = 无法衡量成功 | 立即补充——每个指标必须有基线和目标 |

## 反合理化

| 你可能在想的 | 真相 |
| --- | --- |
| "500 字太少了" | 一页约束是特性不是缺陷。说不清 = 想不清 |
| "先把所有功能列出来" | 全列出来 = 没有优先级。只写 3-5 个核心功能 |
| "权衡不用写" | 不承认权衡 = 不诚实。好的方案承认取舍 |
| "指标以后再定" | 没有指标的方案 = 无法验证的猜测 |
| "技术细节很重要" | 这是给决策者看的，不是给开发者看的 |

## 执行流程

```
1. Read 输入文档，提取关键信息
2. 问题回顾（2-3 句）
   ├── 谁在经历问题？
   ├── 问题是什么？
   └── 为什么现在重要？
3. 方案描述（3-5 句，非技术语言）
4. 关键功能（3-5 个，最小集合）
5. 成功指标（3-5 个，有基线+目标）
6. 权衡（明确不做什么+原因）
7. 风险和缓解
8. 下一步（具体行动+负责人+日期）
9. 按受众调整表达
   ├── 高管 → 聚焦 ROI 和战略价值
   ├── 团队 → 聚焦范围和影响
   └── 客户 → 聚焦价值和差异化
10. Iron Law 检查
    ├── ≤ 500 字？
    ├── 有"不做什么"？
    └── 指标有基线和目标？
```

## 参数

| 参数 | 类型 | 必需 | 默认值 |
| --- | --- | --- | --- |
| 输入文档 | 文件路径 | 是 | — |
| 受众 | 高管/团队/客户 | 否 | 高管 |

## Hard Bans（硬禁令）

以下模式在所有保真度级别均禁止出现。检测到时立即修正。

| # | 禁止模式 | 修正动作 |
| --- | --------- | -------- |
| HB-1 | 方案概述超过 500 字（= 不是一页纸了） | 压缩至 <=500 字，详细内容指向完整 PRD |
| HB-2 | 没有标注来源的定量断言 | 补充数据来源，或改为定性描述并标注 [待验证] |
| HB-3 | 取舍只有优势没有代价 | 每个取舍必须包含"放弃了什么" |
| HB-4 | 无明确受众声明 | 补充受众（高管/团队/客户），调整用语和细节深度 |
| HB-5 | 缺少明确的下一步行动 | 补充具体行动项和负责人 |

## 交付前检查

- [ ] 产出 ≤ 500 字
- [ ] 包含问题/方案/指标/风险/下一步
- [ ] 受众匹配表达方式
- [ ] 关键功能 ≤ 5 个（无 nice-to-haves）
- [ ] 成功指标有基线+目标+时间
- [ ] 明确说明了"不做什么"
- [ ] 下一步有具体行动+负责人

## 产出格式

```markdown
## 方案概要: [方案名]

### 问题（2-3 句）
[解决什么问题，影响谁，为什么现在重要]

### 方案（3-5 句）
[核心方案是什么，与现状的区别，关键用户体验变化]

### 关键功能（3-5 个）
1. **[功能1]**: [简要描述]
2. **[功能2]**: [简要描述]

### 成功指标
| 指标 | 当前 | 目标 | 时间 |
| --- | --- | --- | --- |

### 权衡
- **[不做什么]**: [为什么]

### 风险与缓解
| 风险 | 概率 | 影响 | 缓解 |
|------|------|------|------|

### 下一步
1. [行动项] - [负责人] - [日期]
```

## 常见错误

| 错误 | 后果 | 正确做法 |
| --- | --- | --- |
| 太技术化 | 决策者看不懂 | 用非技术语言，聚焦用户体验 |
| 太多功能 | 一页放不下 | 只写 3-5 个核心功能 |
| 无权衡 | 假装能做所有事 | 诚实承认不做什么 |
| 模糊指标 | 无法衡量成功 | "DAU 从 1 万到 1.5 万" 而非 "提升参与度" |
| 直接跳方案 | 缺少问题上下文 | 先说问题再说方案 |
