# Hard Bans（硬禁令）— pm-prd

以下模式在所有保真度级别均禁止出现。检测到时立即修正。

| # | 禁止模式 | 修正动作 |
| --- | --------- | -------- |
| HB-1 | 风险缓解措施为"持续监控"、"加强沟通"等不可执行描述 | 替换为：谁在什么时间点检查什么指标，达到什么阈值触发什么动作 |
| HB-2 | User Journey/流程中存在无对应功能的步骤（Magic Step） | 补充该步骤对应的功能需求，或从 Journey 中移除 |
| HB-3 | 没有标注来源的定量断言 | 补充数据来源，或改为定性描述并标注 [待验证] |
| HB-4 | 验收标准缺失前置条件 | 补充具体的触发状态和前置条件 |
| HB-5 | In Scope / Out Scope 边界含"视情况"/"待定"/"TBD" | 强制二选一：明确列入 In 或 Out，不确定的列入 Future 并标注决策条件 |
