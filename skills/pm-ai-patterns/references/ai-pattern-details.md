# AI 模式详解

> 6 大 AI 产品特性模式的设计决策指南。

---

## 1. AI 特性模式分类

| 模式 | 描述 | 典型场景 | 用户价值 |
|------|------|----------|----------|
| Completion | 补全用户正在做的事 | 代码补全、写作建议 | 减少输入，加速完成 |
| Generation | 从头生成新内容 | 报告生成、创意文案 | 降低创作门槛 |
| Classification | 自动分类和标注 | 垃圾邮件、情感分析 | 减少人工判断 |
| Extraction | 从非结构化数据提取信息 | 文档摘要、数据提取 | 结构化信息 |
| Recommendation | 个性化推荐 | 内容推荐、商品推荐 | 发现相关内容 |
| Conversation | 自然语言交互 | AI 助手、客服机器人 | 降低使用门槛 |

## 2. 设计决策树

### 选择输入模式

```
用户技术能力？
├── 技术用户 → Prompt 输入（自由度高）
├── 非技术用户 → 表单引导（降低门槛）
└── 编辑器场景 → 上下文感知（零操作）

内容复杂度？
├── 纯文本 → 文本输入
├── 含图片/文件 → 多模态输入
└── 结构化数据 → 表单 + API
```

### 选择输出模式

```
输出长度？
├── 短文本（< 200 字）→ 直接输出
├── 长文本（> 200 字）→ 流式输出
└── 结构化数据 → JSON/表格

信息密度？
├── 单一结论 → 直接展示
├── 多维度 → 渐进展示（摘要 → 详情）
└── 批量结果 → 列表 + 筛选
```

### 选择交互模式

```
任务复杂度？
├── 简单查询 → 单轮问答
├── 复杂任务 → 多轮对话
├── 高风险决策 → 人在回路
└── 批量低风险 → 自主执行

错误代价？
├── 低（可撤销）→ AI 自主 + 用户确认
├── 中（有影响）→ AI 建议 + 用户选择
└── 高（不可逆）→ 人在回路 + 双重确认
```

## 3. 评估框架

### AI 功能评估指标

| 维度 | 指标 | 衡量方式 |
|------|------|----------|
| 准确性 | 准确率/精确率/召回率 | 标注数据评估 |
| 用户接受度 | 接受率 | AI 建议被接受的比例 |
| 效率提升 | 时间节省 | 有/无 AI 完成任务的时间对比 |
| 用户满意度 | NPS/CSAT | 专项调研 |
| 错误影响 | 错误率 × 影响度 | 错误次数 × 每次影响的用户数 |

### A/B 测试设计

| 要素 | 说明 |
|------|------|
| 对照组 | 无 AI 功能的版本 |
| 实验组 | 有 AI 功能的版本 |
| 主要指标 | 用户接受率 或 任务完成率 |
| 次要指标 | 时间、满意度、错误率 |
| 样本量 | 根据最小可检测效应计算 |
| 运行时间 | ≥ 2 周（覆盖周期性变化） |

## 4. 常见陷阱

| 陷阱 | 表现 | 解决方案 |
|------|------|----------|
| 魔法按钮思维 | "AI 会自动解决" | 明确 AI 的输入、处理、输出边界 |
| 忽略失败模式 | 只考虑 AI 成功的情况 | 设计 AI 失败时的降级体验 |
| 过度自动化 | 用户失去控制感 | 关键节点保留人类决策 |
| 冷启动问题 | 新用户 AI 推荐不准 | 设计兜底策略（热门/编辑推荐） |
| 反馈闭环缺失 | AI 不学习不改进 | 设计显式和隐式反馈收集 |
