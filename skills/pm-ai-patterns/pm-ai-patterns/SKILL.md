**输出**: 匹配的 AI 产品模式建议

## Iron Law

| 铁律 | 违反后果 |
| --- | --- |
| AI 边界必须明确（能做什么/不能做什么） | 用户期望失控——明确限制 |
| 用户必须有编辑/拒绝 AI 建议的能力 | AI 独裁——加入人类控制点 |
| 每个 AI 功能必须有评估指标 | 无法衡量好坏——定义成功标准 |

## 反合理化

| 你可能在想的 | 真相 |
| --- | --- |
| "AI 什么都能做" | AI 有明确边界。不承认边界 = 过度承诺 = 用户失望 |
| "用户会自己学会" | AI 功能不引导 = 用户不会用 = 功能浪费 |
| "先上线再优化" | 没有评估指标的 AI = 无法优化的 AI |
| "不需要反馈机制" | 没有反馈 = AI 无法改进 = 越来越差 |
| "置信度不重要" | 不展示置信度 = 用户盲信 AI = 高风险场景翻车 |

## 模式库

### 输入模式

| 模式 | 描述 | 适用场景 |
| --- | --- | --- |
| Prompt 输入 | 自由文本输入 | 通用 AI 助手 |
| 表单引导 | 结构化表单 → Prompt | 非技术用户 |
| 上下文感知 | 自动读取上下文 | 编辑器/IDE 集成 |
| 多模态输入 | 文本+图片+文件 | 复杂内容创作 |

### 输出模式

| 模式 | 描述 | 适用场景 |
| --- | --- | --- |
| 流式输出 | 逐字显示 | 长文本生成 |
| 结构化输出 | JSON/表格 | 数据提取 |
| 批量输出 | 多个结果 | 批量处理 |
| 渐进展示 | 摘要→详情 | 信息密集场景 |

### 交互模式

| 模式 | 描述 | 适用场景 |
| --- | --- | --- |
| 单轮问答 | 一次输入一次输出 | 简单查询 |
| 多轮对话 | 上下文连续对话 | 复杂任务 |
| 人在回路 | AI建议+人确认 | 高风险决策 |
| 自主执行 | AI 全自动 | 低风险批量任务 |

### 置信度展示

| 模式 | 描述 | 适用场景 |
| --- | --- | --- |
| 显式置信度 | 显示百分比 | 数据分析 |
| 来源引用 | 标注信息来源 | 知识问答 |
| 不确定性表达 | "我不确定，但..." | 通用 |
| 多方案展示 | 给多个可选答案 | 开放性任务 |

## 执行流程

```
1. 理解用户场景/产品描述
2. 匹配输入模式 + 输出模式 + 交互模式
3. 产出模式建议 + 适用理由 + 实现注意事项
```

## Hard Bans（硬禁令）

以下模式在所有保真度级别均禁止出现。检测到时立即修正。

| # | 禁止模式 | 修正动作 |
| --- | --------- | -------- |
| HB-1 | 出现超出该保真度级别的视觉元素（线框含品牌色/原型不可交互） | 回到当前保真度的约束范围 |
| HB-2 | 页面/屏幕没有目的声明 | 补充"这个页面让用户完成什么"后继续 |
| HB-3 | AI 功能没有用户控制点（输出用户无法编辑/拒绝） | 添加人类控制点（编辑/拒绝/重新生成） |
| HB-4 | 只展示正常状态，忽略空状态/加载/错误 | 每个关键组件补充异常状态 |
| HB-5 | 页面间无导航路径（孤立页面） | 补充完整的用户路径：入口→操作→目标 |

## 常见错误

| 错误 | 后果 | 正确做法 |
| --- | --- | --- |
| 过度承诺 AI 能力 | 用户失望 | 明确限制和边界 |
| 无编辑能力 | 用户被迫接受差结果 | 允许编辑和重新生成 |
| 无反馈机制 | AI 无法改进 | 收集用户反馈闭环 |
| 单一交互模式 | 不适配场景 | 根据场景选择合适的交互模式 |
| 忽略置信度 | 用户盲信 AI | 在高风险场景展示置信度 |
