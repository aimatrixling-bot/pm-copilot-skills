# PM 方法论推理骨架

PM Copilot 在 Expert 模式下根据产品类型自动选择方法论骨架。

## 1. Working Backwards（逆向工作法）

**适用**：消费者产品、平台型产品、2C 场景

**推理步骤**：
1. 写 Press Release（面向未来用户的产品发布公告）
2. 从 PR 提炼核心假设
3. 列出 FAQ（常见问题和担忧）
4. 定义 Customer Experience（用户旅程）
5. 从 PR 反向推导 MVP 需求

**输出结构**：
```markdown
## Press Release
### 一句话概述
[产品在发布时会怎么描述自己]

### 问题陈述
[用户今天遇到的痛点]

### 解决方案
[产品如何解决这个痛点]

### 核心假设
1. [假设 1]
2. [假设 2]

### FAQ
| 问题 | 回答 |
|------|------|
```

## 2. Jobs To Be Done (JTBD)

**适用**：工具型产品、B2B、功能优化

**推理步骤**：
1. 定义 Job Statement：当 [情境]，我想 [动机]，以便 [期望结果]
2. 识别当前解决方案的不足
3. 定义期望的更好方案
4. 从 Job 推导功能需求
5. 定义成功指标

**输出结构**：
```markdown
## Job Statement
当我 [情境]，我想要 [动机]，以便 [期望结果]

## 当前方案痛点
1. [痛点 1]

## 期望体验
[描述理想的完成路径]

## 功能需求
| Job 步骤 | 对应功能 | 优先级 |
|----------|----------|--------|
```

## 3. Lean Canvas

**适用**：新产品探索、创业型项目、0→1 阶段

**推理步骤**：
1. 问题（Top 3 用户痛点）
2. 客户群体（目标用户画像）
3. 独特价值主张（一句话说清为什么选你）
4. 解决方案（对应每个痛点的功能）
5. 渠道（如何触达用户）
6. 收入模式
7. 成本结构
8. 关键指标
9. 不公平优势

**输出结构**：
```markdown
## Lean Canvas

| 问题 | 解决方案 | 关键指标 | 不公平优势 |
|------|----------|----------|------------|
| 1.   |          |          |            |
| 2.   |          |          |            |
| 3.   |          |          |            |

| 客户群体 | 渠道 | 收入模式 | 成本结构 |
|----------|------|----------|----------|
|          |      |          |          |

## 独特价值主张
[一句话]
```

## 4. 假设驱动开发

**适用**：实验性功能、增长型项目、数据驱动场景

**推理步骤**：
1. 列出关键假设（Belief）
2. 为每个假设设计验证方法
3. 定义最小化验证（ cheapest test）
4. 设定通过/失败标准
5. 规划实验序列

**输出结构**：
```markdown
## 假设列表

| # | 假设 | 信心 | 验证方法 | 通过标准 | 成本 |
|---|------|------|----------|----------|------|
| 1 |      | 高   |          |          |      |
| 2 |      | 中   |          |          |      |
| 3 |      | 低   |          |          |      |

## 实验计划
### 实验 1: [假设 #]
- 方法：
- 样本量：
- 时间：
- 决策规则：
```

## 方法论选择决策树

```
产品类型？
├── 消费者产品 / 2C → Working Backwards
├── 工具型 / B2B / 功能优化 → JTBD
├── 新产品 / 0→1 → Lean Canvas
├── 实验 / 增长 → 假设驱动
└── 不确定 → 询问用户偏好
```
