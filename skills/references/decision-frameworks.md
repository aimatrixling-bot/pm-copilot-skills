# 决策框架对比

## ADR (Architecture Decision Record)

**适用**：技术决策、架构选择、不可逆决策

**模板**：
```markdown
# ADR-[编号]: [决策标题]

## 状态
[提议 | 已接受 | 已废弃 | 已替代]

## 背景
[为什么需要做这个决策]

## 决策
[我们选择了什么]

## 理由
[为什么这样选择]

## 备选方案
| 方案 | 优点 | 缺点 | 评分 |
|------|------|------|------|

## 影响
[这个决策带来的正面和负面影响]

## 开放问题
[还有什么需要后续跟进的]
```

## RAPID 冏策框架

**适用**：团队决策、跨部门协作、流程类决策

**角色定义**：
- **R**ecommend（推荐者）：收集信息，提出建议
- **A**gree（同意者）：必须同意才能推进
- **P**erform（执行者）：执行决策
- **I**nput（输入者）：提供意见但不做最终决定
- **D**ecide（决策者）：最终拍板

**模板**：
```markdown
# RAPID: [决策主题]

## 决策者（D）
[谁最终决定]

## 推荐者（R）
[谁提出建议]

## 同意者（A）
[谁必须同意]

## 输入者（I）
[谁提供意见]

## 执行者（P）
[谁负责执行]

## 决策内容
[具体决定]

## 时间线
[决策和执行的截止日期]
```

## 2x2 决策矩阵

**适用**：优先级排序、资源分配、策略选择

**模板**：
```markdown
# 2x2 矩阵: [维度 X] × [维度 Y]

## 高 [Y] / 低 [X]
[示例项目]

## 高 [Y] / 高 [X]
[示例项目]

## 低 [Y] / 低 [X]
[示例项目]

## 低 [Y] / 高 [X]
[示例项目]

## 优先行动
- 立即做（高 Y 高 X）：
- 投资做（高 Y 低 X）：
- 快速做（低 Y 高 X）：
- 暂缓（低 Y 低 X）：
```

## Weighted Scoring（加权评分）

**适用**：多维度对比、量化决策、方案选型

**模板**：
```markdown
# 加权评分: [决策主题]

## 评分维度
| 维度 | 权重 | 方案 A | 方案 B | 方案 C |
|------|------|--------|--------|--------|
|      |      | 原始   | 加权   | 原始   | 加权   | 原始   | 加权   |
|      | %    | 1-5    | W×S    | 1-5    | W×S    | 1-5    | W×S    |

## 总分
| 方案 | 总加权分 |
|------|----------|

## 推荐
[基于评分的推荐 + 定性分析补充]
```

## 框架选择指南

```
决策场景？
├── 技术架构选择 → ADR
├── 团队/流程决策 → RAPID
├── 优先级/资源分配 → 2x2 矩阵
├── 多方案量化对比 → Weighted Scoring
├── 不确定 → 询问用户偏好
└── 简单决策 → 直接推理，不使用框架
```
