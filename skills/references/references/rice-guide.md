# RICE 评分指南

## 维度说明

### Reach（影响范围）
在给定时间范围内，该功能会影响多少用户/客户？

| 评分 | 含义 |
|------|------|
| 1-100 | 每季度预估影响的用户数 |
| 或百分比 | 占总用户群的比例 × 总用户数 |

### Impact（影响程度）
对每个受影响用户的影响有多大？

| 评分 | 含义 | 说明 |
|------|------|------|
| 3 | Massive | 核心体验提升，用户会主动推荐 |
| 2 | High | 显著改善，用户能明显感知 |
| 1 | Medium | 有一定改善，但不是关键 |
| 0.5 | Low | 微小改善，锦上添花 |
| 0.25 | Minimal | 几乎无感知 |

### Confidence（信心度）
你对 Reach 和 Impact 的估计有多确定？

| 评分 | 含义 | 说明 |
|------|------|------|
| 100% | High | 有定量数据支撑 |
| 80% | Medium | 有定性反馈或竞品数据 |
| 50% | Low | 纯直觉或假设 |

### Effort（工作量）
完成该功能需要多少人力？

| 评分 | 含义 |
|------|------|
| 人月 | 一个人多少个月（可以是小数如 0.5） |
| 或人周 | 一个人多少周 |

## 计算公式

```
RICE Score = (Reach × Impact × Confidence) / Effort
```

## 评分引导

### 防止"全打高分"倾向

| 你可能在想的 | 真相 |
| --- | --- |
| "每个功能都很重要" | 资源有限，必须区分优先级 |
| "Impact 全打 3 分" | 只有真正改变用户行为的功能才配 3 分 |
| "Effort 很难估计" | 给一个范围（如 1-2 人月），取中间值 |
| "Confidence 全打 80%" | 如果你没有数据，就是 50% |

### 评分校准

- **Impact ≥ 2** 的功能应该 ≤ 总功能的 30%
- **Effort ≤ 0.5** 的功能可以优先考虑（快速胜利）
- **Confidence = 50%** 的功能需要先做验证实验

