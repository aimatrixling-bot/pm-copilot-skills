## 1. Hook 模型（核心框架）

**来源**: Introduction, p6-11 | **PM 相关度**: ⭐⭐⭐

```
Hook 模型四阶段循环

         ┌──────────────────────────────────────────┐
         │                                          │
         │    ┌───────────┐                         │
         │    │  Trigger  │ ← 触发器                 │
         │    │  （触发）  │   外部：通知/邮件/广告    │
         │    └─────┬─────┘   内部：情绪/习惯/痛点    │
         │          │                               │
         │    ┌─────▼─────┐                         │
         │    │  Action   │ ← 行为                   │
         │    │  （行动）  │   最小努力完成核心行为     │
         │    └─────┬─────┘   B = MAT (动机×能力×触发) │
         │          │                               │
         │    ┌─────▼─────┐                         │
         │    │  Variable │ ← 多变酬赏               │
         │    │  Reward   │   部落/猎物/自我          │
         │    │  （酬赏）  │   关键：不可预测性         │
         │    └─────┬─────┘                         │
         │          │                               │
         │    ┌─────▼─────┐                         │
         │    │ Investment│ ← 投入                   │
         │    │  （投入）  │   数据/内容/联系/信誉/技能 │
         │    └───────────┘   加载下一个触发器         │
         │                                          │
         └──────────────────────────────────────────┘

Hook ≠ 成瘾：
├── Hook 是帮用户养成好习惯
├── 成瘾是损害用户利益
└── 道德测试：自己用吗？改善用户生活吗？
```

## 2. 习惯区域矩阵

**来源**: Ch 1, p12-22 | **PM 相关度**: ⭐⭐⭐

```
习惯区域：何时适合做习惯产品？

感知价值
   ↑
   │  ┌─────────────┐
   │  │  习惯区域    │ ← 高频 + 高价值 = 习惯形成机会
   │  │  (Habit Zone)│
   │  └─────────────┘
   │
   │  ┌─────────────┐
   │  │  维生素      │ ← 高价值但低频 = 需要外部触发器
   │  │  (Vitamin)   │    例：旅游保险、房产交易
   │  └─────────────┘
   ├────────────────────────────→ 使用频率

   │  ┌─────────────┐
   │  │  止痛药      │ ← 低价值但高频 = 习惯一旦形成难改变
   │  │  (Painkiller)│    例：搜索引擎
   │  └─────────────┘

   │  ┌─────────────┐
   │  │  白菜        │ ← 低频 + 低价值 = 不适合做习惯产品
   │  │  (Cabbage)   │
   │  └─────────────┘

习惯的商业价值：
├── 提升客户生命周期价值（CLTV）
├── 增强定价灵活性
├── 加速增长（口碑传播）
└── 构筑竞争壁垒（"切换成本" = 用户习惯）
```

## 3. 触发器模型

**来源**: Ch 2, p23-32 | **PM 相关度**: ⭐⭐⭐

```
触发器分类

外部触发器（External Triggers）：
├── 付费触发器（Paid）：广告、搜索引擎营销
├── 病毒触发器（Earned）：口碑、PR、社交媒体分享
├── 关系触发器（Relational）：邀请、推荐
└── 自有触发器（Owned）：推送通知、邮件、应用内提示

内部触发器（Internal Triggers）：
├── 驱动力：情绪（无聊、焦虑、孤独、不确定）
├── 来源：用户内在的心理状态
├── 发现方法：5 Whys 方法
└── 特点：一旦形成，用户会自我触发

触发器演进路径：
Paid → Earned → Relationship → Owned → Internal
（付费→赢得→关系→自有→内部）

最终目标：让内部触发器驱动用户行为，
外部触发器逐步减少依赖
```

## 4. Fogg 行为模型（B=MAP）

**来源**: Ch 3, p33-49 | **PM 相关度**: ⭐⭐⭐

```
Fogg 行为模型

B = M × A × T

Behavior = Motivation × Ability × Trigger

行为发生条件：三个要素同时满足

动机 (Motivation)：
├── 追求快乐 / 逃避痛苦
├── 追求希望 / 逃避恐惧
└── 追求社交认同 / 逃避排斥

能力 (Ability) —— 六大简洁性要素：
├── 时间（Time）：完成行为需要多长时间？
├── 金钱（Money）：需要花费多少？
├── 体力（Physical Effort）：需要多少体力劳动？
├── 脑力（Brain Cycles）：需要多少认知努力？
├── 社会偏差（Social Deviance）：行为是否符合社会规范？
└── 非常规性（Non-Routine）：行为是否打破了常规？

设计原则：
├── 提高能力比提高动机更容易
├── 减少"摩擦"是最有效的策略
└── 简洁 = 更高的行为完成率

四种关键启发式：
├── 稀缺效应（Scarcity）
├── 框架效应（Framing）
├── 锚定效应（Anchoring）
└── 禀赋进度（Endowed Progress）
```

## 5. 多变酬赏（Variable Reward）

**来源**: Ch 4, p50-67 | **PM 相关度**: ⭐⭐⭐

```
三类多变酬赏

┌────────────────────────────────────────────────┐
│  部落酬赏 (Tribal Rewards)                      │
│  └── 社交认同、接纳、归属感                       │
│      例：Facebook 点赞、知乎赞同、游戏排行榜       │
├────────────────────────────────────────────────┤
│  猎物酬赏 (Hunt Rewards)                        │
│  └── 资源、信息、物质奖励                         │
│      例：Pinterest 无限滚动、Twitter 刷新、淘宝浏览│
├────────────────────────────────────────────────┤
│  自我酬赏 (Self Rewards)                        │
│  └── 成就感、掌控感、个人成长                      │
│      例：Duolingo 连续打卡、游戏通关、运动记录     │
└────────────────────────────────────────────────┘

关键原则：
├── 酬赏必须是"多变的"（不可预测）—— 这是多巴胺的驱动力
├── 固定酬赏会削弱习惯形成（Skinner 鸽子实验）
├── 有限的多变性 vs 无限的多变性
│   ├── 有限：内容最终会被耗尽（Netflix）
│   └── 无限：用户生成内容确保永不重复（YouTube）
└── 避免将最终目标与可变酬赏绑定（"用户彩票"陷阱）
    └── 用户应该因为行为本身而满足，而非因为外部奖励
```

## 6. 投入模型（Investment）

**来源**: Ch 5, p68-79 | **PM 相关度**: ⭐⭐⭐

```
投入：为下一次 Hook 做准备

核心机制："先给酬赏，再求投入"
├── 投入不等于沉没成本
├── 投入加载下一个触发器
└── 投入增加用户的切换成本

五类存储价值：
┌────────────────────────────────────────────────┐
│  1. 内容（Content）                             │
│     └── 用户创作的内容：帖子、评论、笔记          │
├────────────────────────────────────────────────┤
│  2. 数据（Data）                                │
│     └── 用户的行为数据：偏好、历史、设置          │
├────────────────────────────────────────────────┤
│  3. 关注者/联系（Followers/Connections）          │
│     └── 社交关系网络                             │
├────────────────────────────────────────────────┤
│  4. 信誉（Reputation）                          │
│     └── 声誉系统：积分、等级、徽章               │
├────────────────────────────────────────────────┤
│  5. 技能（Skill）                               │
│     └── 用户习得的使用技能                       │
└────────────────────────────────────────────────┘

心理驱动力：
├── 宜家效应（IKEA Effect）：人们更重视自己投入劳动的事物
├── 渐进承诺（Progressive Commitment）：小投入→大投入
└── 认知失调（Cognitive Dissonance）：为过去的投入寻找合理性
```

## 7. 操控矩阵（道德框架）

**来源**: Ch 6, p80-86 | **PM 相关度**: ⭐⭐

```
操控矩阵（Manipulation Matrix）

                改善用户生活
                     │
          ┌──────────┼──────────┐
          │                     │
    是    │  促进者     │  推销者  │
  自己用   │  Facilitator│ Peddler │
          │  (好习惯)    │ (坏习惯) │
          ├──────────┼──────────┤
          │                     │
    否    │  娱乐者     │  贩子    │
  不用    │  Entertainer│ Dealer  │
          │  (无害)      │ (有害)  │
          │                     │
                不改善用户生活

PM 道德标准：
├── 做促进者（自己用 + 改善用户生活）
├── 避免"用户彩票"——不要用可变酬赏操纵用户
└── 习惯产品的责任在于确保习惯是健康的
```

## 8. 习惯测试三步法

**来源**: Ch 8, p92-98 | **PM 相关度**: ⭐⭐

```
习惯测试流程

Step 1: 识别（Identify）
├── 谁是习惯用户？（5% 法则：5% 的用户使用频率 ≥ 其他用户的 2 倍）
├── 他们的行为路径是什么？
└── 他们有多少是通过外部触发器进入的？

Step 2: 编码（Codify）
├── 将习惯用户的行为路径转化为可追踪的事件
├── 画出用户从触发器到酬赏的完整路径
└── 找到 Hook 循环的薄弱环节

Step 3: 修改（Modify）
├── 优化触发器
├── 减少行动的摩擦
├── 增强酬赏的多变性
└── 增加有意义的投入

发现习惯形成机会的四个方向：
├── 1. 自我观察：你自己的哪些行为是习惯性的？
├── 2. 新兴行为：新技术催生的新习惯机会
├── 3. 使能技术：新技术让旧行为变得更容易
└── 4. 界面变革：交互方式的变化创造新机会
```
