## 1. 函数方法论：样本 + 特征 + 评估

**来源**: Ch 3-4, p60-142 | **PM 相关度**: ⭐⭐⭐

```
函数方法论 3+1 要素

策略 = f(样本, 特征, 评估) + 基准策略

┌─────────────────────────────────────────────┐
│              策略函数 f(X)                     │
│                                              │
│   输入            处理           输出          │
│  ┌──────┐     ┌─────────┐     ┌──────┐     │
│  │ 样本  │────→│ 策略模型 │────→│ 评估  │     │
│  └──────┘     └─────────┘     └──────┘     │
│       ↑                           ↑         │
│   数据基础                    效果衡量      │
│                                              │
│   ┌─────────────────────┐                   │
│   │  +1: 基准策略        │                   │
│   │  (兜底/对比基准)     │                   │
│   └─────────────────────┘                   │
└─────────────────────────────────────────────┘
```

### 三要素详解

| 要素 | 核心问题 | 关键方法 |
|------|---------|---------|
| **样本** | 用什么数据训练/评估？ | 随机采样、分层采样、正负样本比例、时间切分 |
| **特征** | 用什么信息做决策？ | 特征选择（过滤法/包裹法/嵌入法）、特征工程 |
| **评估** | 怎么衡量策略好坏？ | 离线评估（AUC/NDCG/CTR）、在线评估（A/B测试） |

**PM 对应**：这是策略 PM 最核心的方法论框架，所有策略设计都可以用这个框架分析。

## 2. 召回-排序架构

**来源**: Ch 5, p204-211 | **PM 相关度**: ⭐⭐⭐

```
召回-排序架构（搜索/推荐/广告通用）

          ┌──────────────┐
          │    召回层      │ ← 从海量候选中快速筛选千级别候选
          │  (Recall)     │    多路召回：热门、协同过滤、内容匹配...
          └──────┬───────┘
                 │
          ┌──────▼───────┐
          │    粗排层      │ ← 用轻量模型快速排序
          │  (Pre-rank)   │
          └──────┬───────┘
                 │
          ┌──────▼───────┐
          │    精排层      │ ← 用复杂模型精确打分
          │  (Rank)       │    深度学习、特征交叉...
          └──────┬───────┘
                 │
          ┌──────▼───────┐
          │    策略层      │ ← 业务规则、多样性、去重
          │  (Strategy)   │
          └──────────────┘
```

**三大场景对比**：

| 维度 | 搜索 | 推荐 | 广告 |
|------|------|------|------|
| **核心目标** | 满足信息需求 | 提升用户兴趣 | 最大化商业价值 |
| **用户意图** | 明确（查询词） | 隐含（行为推断） | 混合 |
| **评估指标** | NDCG、DCG | CTR、留存率 | CTR、CVR、ROI |
| **核心挑战** | 相关性 | 多样性+惊喜度 | eCPM优化 |

## 3. 基准策略

**来源**: Ch 3, p85-96 | **PM 相关度**: ⭐⭐⭐

```
三大基准策略

1. 热门排行策略
   └── 按历史热度排序
       优点：简单、有兜底效果
       缺点：无法个性化
       适用：冷启动、新用户

2. 最近行为策略
   └── 基于用户近期行为推荐
       优点：捕捉短期兴趣
       缺点：信息茧房
       适用：个性化推荐的起点

3. 业务规则策略
   └── 基于业务专家经验设定规则
       优点：可控、可解释
       缺点：依赖人工、难以扩展
       适用：高价值场景、合规要求
```

**核心思想**：基准策略是策略迭代的起点，任何新策略都必须先跑赢基准策略。

## 4. 经济学方法论

**来源**: Ch 6, p212-264 | **PM 相关度**: ⭐⭐⭐

```
经济学方法论两大问题类型

1. 定价问题 (Pricing)
   └── 给商品/服务定什么价格？
       方法：边际分析、价格弹性、动态定价
       案例：Airbnb 动态定价

2. 匹配/调度问题 (Matching/Dispatch)
   └── 如何最优地分配资源？
       方法：博弈论、拍卖理论、优化算法
       案例：美团骑手派单

关键经济学概念：
├── 理性人假设：用户追求效用最大化
├── 边际分析：额外一单位的收益 vs 成本
├── 博弈论：多方参与的策略互动
└── 拍卖理论：资源的最优分配机制
```

**函数方法论 vs 经济学方法论**：

| 维度 | 函数方法论 | 经济学方法论 |
|------|-----------|-------------|
| 适用场景 | 排序、分类、预测 | 定价、匹配、调度 |
| 核心方法 | 机器学习 | 优化理论 |
| 数据依赖 | 高（需要大量样本） | 低（可以用规则） |
| 可解释性 | 较低 | 较高 |

## 5. ML 模型速查（PM 视角）

**来源**: Ch 7, p267-358 | **PM 相关度**: ⭐⭐

| 模型 | 核心思想 | 优势 | 劣势 | PM 适用场景 |
|------|---------|------|------|-----------|
| **决策树** | 递归分割 | 可解释、无需特征缩放 | 容易过拟合 | 需要向业务方解释时 |
| **K-NN** | 找最近的K个邻居 | 简单直观 | 计算量大 | 小规模数据、快速原型 |
| **SVM** | 找最大间隔超平面 | 高维表现好 | 参数调优复杂 | 二分类问题 |
| **朴素贝叶斯** | 贝叶斯定理+条件独立 | 计算快、小数据好 | 特征独立假设 | 文本分类、垃圾邮件 |
| **逻辑回归** | Sigmoid函数映射概率 | 输出概率、易训练 | 线性边界 | CTR预估、基线模型 |

## 6. 深度学习与三要素

**来源**: Ch 8, p359-401 | **PM 相关度**: ⭐⭐

```
深度学习对三要素的扩展

特征：
└── 自编码器（Autoencoder）自动学习特征表示
    无监督预训练 → 微调
    解决人工特征工程的瓶颈

样本：
└── 迁移学习（Transfer Learning）解决样本不足
    大模型预训练 → 小数据微调
    解决冷启动问题

评估：
└── 多任务学习（Multi-task Learning）同时优化多个目标
    共享底层特征 → 各任务独立输出
    解决多目标优化问题
```

## 9. RFM 模型

**来源**: Ch 5, p149-154 | **PM 相关度**: ⭐⭐

```
RFM 模型（客户价值分析）

R (Recency): 最近一次消费距今天数
F (Frequency): 消费频次
M (Monetary): 消费金额

客户分层：
├── 高价值客户：R近 + F高 + M高
├── 流失风险客户：R远 + F低
├── 成长型客户：R近 + F低 + M低
└── 沉睡客户：R远 + F低 + M低

营销投放策略：
├── 对高价值客户：维持策略
├── 对流失风险客户：唤醒策略
└── 对成长型客户：促活策略
```
