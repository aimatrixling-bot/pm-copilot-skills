## 1. 数据产品三层分类模型

**来源**: Ch 1, p20-55 | **PM 相关度**: ⭐⭐⭐

```
数据产品
├── 用户数据产品 — 面向终端用户的数据服务
│   例: 数据可视化报表、自助分析工具、个性化推荐
│
├── 商用数据产品 — 面向外部客户的数据变现
│   例: Domo、Salesforce Analytics、数据交易平台
│
└── 企业数据产品 — 面向内部业务的数据基础设施
    例: 数据中台、指标平台、标签平台、A/B测试系统
```

**PM 对应**：PM OS 本质上是一个"企业数据产品"，服务于 PM 的日常工作。

## 2. 数据 PM 能力模型

**来源**: Ch 1, p55-68 | **PM 相关度**: ⭐⭐⭐

三个维度的能力要求：

| 维度 | 内容 | 说明 |
|------|------|------|
| **产品经理能力** | 需求分析、产品设计、项目管理、沟通协调 | 通用 PM 技能 |
| **数据专业能力** | SQL、数据分析、指标体系、A/B 测试、数据建模 | 数据领域硬技能 |
| **软能力** | 业务理解、跨部门协作、向上管理、技术沟通 | 决定天花板的能力 |

**数据 PM 三种类型**：

| 类型 | 定位 | 典型产出 |
|------|------|---------|
| **平台型** | 建设数据基础设施 | 数据中台、指标平台、标签平台 |
| **应用型** | 利用数据解决具体业务问题 | BI 报表、推荐系统、用户画像 |
| **策略型** | 通过算法驱动业务决策 | 搜索排序、风控策略、定价策略 |

## 3. 数据分析四步法

**来源**: Ch 2, p110-118 | **PM 相关度**: ⭐⭐⭐

```
发现问题 → 定位问题 → 分析问题 → 提出结论
    │           │           │           │
    │           │           │           └── "增减思路"：增加收益 OR 减少损失
    │           │           └── 综合使用多种分析方法
    │           └── 使用全链路/因子分解/影响因子/枚举法
    └── 明确问题（有明确问题 vs 无明确问题）
```

**有价值的数据结论 = 增加收益 + 减少损失**（"增减思路"）

## 4. 四种数据分析方法

**来源**: Ch 2, p119-138 | **PM 相关度**: ⭐⭐⭐

| 方法 | 用途 | 适用场景 |
|------|------|---------|
| **全链路分析** | 发现每个环节的问题 | 漏斗分析、转化率优化 |
| **组成因子分解** | 发现不同部门/项目的问题 | 按业务模块、团队、渠道拆解 |
| **影响因子拆解** | 发现哪个因素造成影响 | 排除干扰因素、找到核心变量 |
| **枚举法** | 快速发现重要客户/商品/流量 | Top N 分析、帕累托法则 |

**来源**: Ch 3, p173-184 | **PM 相关度**: ⭐⭐⭐

| 模型 | 方法 | 适用场景 |
|------|------|---------|

## 6. 数据指标体系框架

**来源**: Ch 6, p244-290 | **PM 相关度**: ⭐⭐⭐

### 指标的构成

```
指标 = 维度 × 汇总方式 × 量度

例: "某城市 7 月 播放总时长"
     维度      汇总方式   量度
    (城市+月份)  (求和)    (分钟数)
```

### 指标分类体系

```
数据指标
├── 按指标类型
│   ├── 绝对指标 — 计数类（PV、UV、订单数）
│   └── 相对指标 — 比率类（转化率、留存率、ARPU）
│
└── 按数据指标类型（本书核心分类）
    ├── 原子指标 — 不可再拆分的度量（呼单量、支付金额）
    ├── 派生指标 — 原子指标 + 修饰词 + 时间周期
    │   ├── 事务型: 衡量业务过程
    │   └── 存量型: 衡量实体状态
    └── 衍生指标 — 复合指标（比率型、比例型、均值型）
```

### 指标体系建设方法

```
1. 明确建设目标（业务目标驱动）
2. 梳理业务过程（业务流程拆解）
3. 定义指标体系（OSM + 分级 + 维度）
4. 设计指标字典（统一口径、命名、计算逻辑）
5. 持续运营迭代（生命周期管理）
```

## 8. 数据中台五步构建法

**来源**: Ch 5, p213-243 | **PM 相关度**: ⭐⭐

```
定战略 → 改组织 → 深业务 → 做统一 → 享服务
   │        │        │        │        │
   │        │        │        │        └── 数据服务化、API化
   │        │        │        └── 统一指标、统一标签、统一口径
   │        │        └── 深入业务场景，理解业务逻辑
   │        └── 调整组织架构，建立协同机制
```

**数据中台四大产品形态**：

| 产品 | 核心功能 |
|------|---------|
| **统一指标平台** | 指标定义、计算、管理 |
| **统一标签平台** | 用户/实体标签管理 |
| **可视化报表平台** | BI 看板、自助分析 |
| **智慧营销平台** | 人群圈选、精准触达 |

## 9. 用户画像标签体系

**来源**: Ch 11, p467-529 | **PM 相关度**: ⭐⭐⭐

### 标签类型

| 类型 | 说明 | 举例 |
|------|------|------|
| **事实标签** | 直接从数据提取 | 注册时间、最近登录、历史订单数 |
| **统计标签** | 聚合计算 | 月均消费、7日活跃天数、品类偏好 |
| **算法标签** | 模型预测 | 流失概率、购买意向、兴趣偏好 |

### 从 0 到 100 构建思路

```
0→1: 确定核心标签（基于业务需求优先级）
     ↓
1→10: 搭建标签生产流程（定义→特征→模型→验收）
     ↓
10→100: 扩展标签体系（增加算法标签、内容标签）
       建立标签生命周期管理
       完善效果验收机制
```

### 标签效果验收四法

1. **算法指标验收** — Precision、Recall、F1-Score
2. **分布验证** — 标签值分布是否符合预期
3. **交叉验证** — 与其他标签交叉分析合理性
4. **抽样评测** — 人工抽样检查标签准确性

## 10. 数据管理三层架构

**来源**: Ch 8, p346-393 | **PM 相关度**: ⭐⭐

```
数据管理
├── 主数据管理（Master Data）
│   四要素: 数据标准、数据质量、数据安全、数据集成
│   核心: 确保核心实体（用户、商品、组织）的唯一性和一致性
│
├── 元数据管理（Metadata）
│   技术/业务/操作元数据三层
│   核心: 知道"有什么数据、在哪里、怎么用"
│
└── 数据服务（Data Service）
    API 服务、指标池服务、可视化模式
    核心: 让数据"好用、易用"
```
