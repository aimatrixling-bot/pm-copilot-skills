## 1. 从业务问题到数据科学任务

**来源**: Ch 2, p43-50 | **PM 相关度**: ⭐⭐⭐

```
业务问题 → 数据科学任务的映射

业务问题类型              对应数据科学任务
├── 分类预测              → 分类 (Classification)
│   例: 用户是否会流失？    例: 逻辑回归、决策树、SVM
├── 数值预测              → 回归 (Regression)
│   例: 下月营收多少？     例: 线性回归
├── 用户分群              → 聚类 (Clustering)
│   例: 客户分几类？       例: K-Means、层次聚类
├── 关联发现              → 关联规则 (Association Rules)
│   例: 购买了A还买什么？   例: Apriori、FP-Growth
├── 排序推荐              → 排序/协同过滤
│   例: 给用户推荐什么？    例: 矩阵分解
└── 文本理解              → 文本挖掘 (Text Mining)
    例: 评论是正面还是负面？ 例: TFIDF + 分类器
```

**核心原则**：PM 的工作是**精确描述业务目标**，数据科学家的工作是选择**合适的技术方案**。

## 3. 期望值框架（Expected Value Framework）

**来源**: Ch 7, p218-228; Ch 11, p301-311 | **PM 相关度**: ⭐⭐⭐

```
期望值 = Σ (概率 × 收益)

决策矩阵示例（客户流失预测）：

                    实际流失    实际不流失
预测流失 + 给优惠     TP: 利润1    FP: 成本1
预测不流失            FN: 损失1    TN: 利润2

期望利润 = P(TP) × 利润1 + P(FP) × 成本1 + P(FN) × 损失1 + P(TN) × 利润2
```

**分析工程（Analytical Engineering）**：
- 将业务问题**分解**为子问题
- 每个子问题对应一个数据科学解决方案
- 将子方案**重组**为完整业务解决方案
- 用期望值框架量化每个决策点的价值

**PM 实践**：这个框架是评估 AI/ML 产品 ROI 的核心工具。

## 4. 模型评估工具箱

**来源**: Ch 7-8, p211-255 | **PM 相关度**: ⭐⭐⭐

| 工具 | 用途 | 适用场景 |
|------|------|---------|
| **混淆矩阵** | TP/FP/FN/TN 全景 | 分类问题的基础评估 |
| **准确率 (Accuracy)** | 正确预测比例 | 类别平衡时使用 |
| **精确率 (Precision)** | 预测为正中实际为正 | 误报代价高时关注 |
| **召回率 (Recall)** | 实际为正中预测为正 | 漏报代价高时关注 |
| **ROC 曲线** | TPR vs FPR 权衡 | 不同阈值下的性能对比 |
| **AUC** | ROC 曲线下面积 | 模型整体排序能力 |
| **Lift 曲线** | 模型 vs 随机的提升倍数 | 营销场景的触达效率 |
| **利润曲线** | 不同阈值下的利润 | 商业决策直接依据 |

**关键洞察**：选择正确的评估指标取决于**业务场景**（不等成本/收益 → 期望值 > 准确率）。

## 5. 过拟合与泛化

**来源**: Ch 5, p135-164 | **PM 相关度**: ⭐⭐⭐

```
过拟合的本质：模型记住了训练数据的噪声，而非学到了真实的模式

                训练误差
                 │  ╲
                 │    ╲
                 │     ╲
                 │      ╲  ← 泛化差距（Overfitting Gap）
   测试误差      │   ·    ╲
                 │    ·     ╲
                 │     ·      ╲____
                 │      ·          ╲_____
                 └──────────────────────── 复杂度
                    最佳点

避免过拟合的方法：
├── 交叉验证 (Cross-Validation)：最常用的泛化能力评估方法
├── 正则化 (Regularization)：限制模型复杂度
├── 提前停止 (Early Stopping)：在验证集性能下降时停止
├── 简化模型：减少参数/特征
└── 更多数据：最根本的解决方案
```

**PM 实践**：当模型在训练集表现好但在生产环境表现差时，首先要怀疑过拟合。

## 6. 有监督 vs 无监督学习

**来源**: Ch 2-3, p48-50, p67-102 | **PM 相关度**: ⭐⭐⭐

| 维度 | 有监督学习 (Supervised) | 无监督学习 (Unsupervised) |
|------|------------------------|--------------------------|
| **输入** | 有标签数据（X + Y） | 无标签数据（只有 X） |
| **目标** | 预测目标变量 | 发现数据中的模式/结构 |
| **典型任务** | 分类、回归 | 聚类、降维、关联规则 |
| **评估方式** | 与真实标签对比 | 主观评估或下游任务效果 |
| **PM 场景** | 流失预测、欺诈检测 | 客户分群、商品推荐、异常发现 |

## 7. 信息增益与决策树

**来源**: Ch 3, p72-102 | **PM 相关度**: ⭐⭐

```
决策树的核心思想：通过递归分割，找到最能区分类别的属性

Information Gain = Entropy(父) - Σ [P(子) × Entropy(子)]

决策树的优点（PM 视角）：
├── 可解释性强：可以清晰展示决策逻辑
├── 不需要特征缩放
├── 能处理数值型和类别型特征
└── 天然支持多分类

决策树的缺点：
├── 容易过拟合
├── 对噪声敏感
└── 边界呈阶梯状（不够平滑）
```

**PM 实践**：当需要向业务方解释模型逻辑时，决策树是最好的选择。

## 8. 贝叶斯思维与证据推理

**来源**: Ch 9, p257-271 | **PM 相关度**: ⭐⭐

```
贝叶斯规则：P(A|B) = P(B|A) × P(A) / P(B)

朴素贝叶斯（Naive Bayes）：
- 假设特征之间条件独立
- 计算简单、速度快
- 在高维稀疏数据（如文本）上表现好

证据提升（Evidence Lift）：
Lift = P(目标|证据) / P(目标)

Lift > 1: 证据增加了目标的概率
Lift < 1: 证据降低了目标的概率
Lift = 1: 证据与目标无关
```

**PM 实践**：Evidence Lift 是评估"某个特征/行为对目标有多大预测力"的直觉性工具。

## 9. 文本表示方法

**来源**: Ch 10, p273-299 | **PM 相关度**: ⭐⭐

```
文本 → 数值表示的演进

Bag of Words → TF (Term Frequency) → TFIDF → N-gram → Topic Models → Embeddings

TFIDF = Term Frequency × Inverse Document Frequency
       (词频)            (逆文档频率)

- TF: 词在文档中出现越多越重要
- IDF: 在所有文档中都出现的词（如"的"）不重要
- TFIDF 平衡了局部重要性和全局稀缺性
```

## 10. 数据科学竞争优势模型

**来源**: Ch 13, p337-351 | **PM 相关度**: ⭐⭐⭐

```
数据科学竞争优势的四个来源：

1. 历史数据优势 (Formidable Historical Advantage)
   └── 长期积累的独特数据资产，竞争对手无法复制

2. 独特的知识产权 (Unique Intellectual Property)
   └── 专有算法、模型、流程

3. 独特的无形资产 (Unique Intangible Collateral Assets)
   └── 品牌声誉、用户网络效应

4. 优秀的数据科学团队 (Superior Data Scientists)
   └── 人才是最重要的竞争壁垒

数据科学成熟度模型：
Level 1: 数据感知 — 开始收集和存储数据
Level 2: 数据驱动 — 用数据分析支持决策
Level 3: 数据科学 — 用预测模型驱动行动
Level 4: 数据创新 — 数据科学创造新的商业价值
```

**Proposal Review Guide（附录 A）**：评估数据科学项目提案的系统检查清单，涵盖业务理解、数据准备、建模、评估、部署五个维度。
