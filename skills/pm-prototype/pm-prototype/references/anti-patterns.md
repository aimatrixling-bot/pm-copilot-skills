# 交互原型反模式参考

> **识别并避免常见的交互原型设计错误**

---

## 视觉反模式

### 1. 通用美学陷阱

**特征**:
- 紫色渐变 + Inter 字体
- 千篇一律的 "SaaS 风格"
- 缺乏品牌识别度

**解决方案**:
- 选择独特的美学基调
- 使用独特的字体组合
- 建立一致的视觉语言

### 2. 过度动效

**特征**:
- 所有元素都在动
- 无意义的旋转、缩放
- 干扰用户注意力

**解决方案**:
- 只动画有意义的交互
- 保持快速 (200-300ms)
- 提供"减少动效"选项

### 3. 颜色混乱

**特征**:
- 使用超过 3 种主色
- 缺乏颜色层次
- 对比度不足

**解决方案**:
- 限制主色数量 (1-2 个)
- 使用颜色系统工具
- 检查 WCAG 对比度

## 结构反模式

### 4. 非语义化 HTML

**反例**:
```html
<div class="button" onclick="submit()">提交</div>
<div class="input">...</div>
```

**正例**:
```html
<button type="submit">提交</button>
<input type="text" required />
```

### 5. 忽略状态

**缺失的状态**:
- 加载状态
- 错误状态
- 空状态
- 禁用状态

**解决方案**:
为每个组件定义所有状态

### 6. 硬编码数据

**反例**:
```javascript
const users = [
  { name: "张伟", score: 95 },
  { name: "李娜", score: 87 }
];
```

**正例**:
```javascript
// 匹配 API 结构
const users = await fetchUsers();
const score = users[0].engagement_score;
```

## 交互反模式

### 7. 忽视键盘导航

**问题**:
- 无法用 Tab 键导航
- 焦点指示不可见
- 无法用键盘执行操作

**解决方案**:
- 确保所有交互可键盘访问
- 清晰的焦点指示
- 支持标准快捷键

### 8. 拥塞界面

**特征**:
- 一次性展示所有信息
- 缺乏视觉层次
- 压倒性设计

**解决方案**:
- 渐进式披露
- 明确视觉层次
- 留白 = 奢华

### 9. 不一致的交互

**问题**:
- 相同功能不同交互
- 按钮位置不固定
- 手势冲突

**解决方案**:
- 建立交互模式库
- 一致的模式应用
- 清晰的手势定义

## 数据反模式

### 10. 假数据

**问题**:
- 使用假用户名、假数字
- 不反映真实数据结构
- 隐藏边界情况

**解决方案**:
- 使用真实数据类型
- 反映真实数据范围
- 包含边界情况 (长文本、特殊字符)

### 11. 忽略错误情况

**缺失**:
- 网络错误处理
- 验证错误处理
- 服务器错误处理

**解决方案**:
- 为所有 API 调用添加错误处理
- 显示友好的错误消息
- 提供恢复选项

### 12. 忽略加载状态

**问题**:
- 数据加载时界面空白
- 无加载指示器
- 用户不知道是否在等待

**解决方案**:
- 添加骨架屏或加载指示器
- 显示加载进度
- 保持界面响应

## 原型陷阱

### 13. 原型即成品

**问题**:
- 原型被认为完成
- 利益相关者期待原型的功能
- 团队停止思考

**防御方案**:
- 明确标注"原型"
- 使用水印/边框
- 沟通原型目的

### 14. 视觉主导功能讨论

**问题**:
- 评审时关注颜色而非功能
- 视觉偏离产品目标

**防御方案**:
- 从产品目标开始
- 将视觉讨论推迟
- 建立优先级: 功能 > 视觉

### 15. 技术债务累积

**问题**:
- 原型代码直接用于生产
- 架构不适合扩展
- 技术债务未记录

**防御方案**:
- 遵循框架最佳实践
- 从开始考虑扩展
- 记录技术决策和债务
