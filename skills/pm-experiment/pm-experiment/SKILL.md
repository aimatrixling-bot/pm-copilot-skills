# 实验设计

<SUBAGENT-STOP>
如果是子代理派发执行此 Skill，直接按下方指令执行，不重新加载入口 Skill。
</SUBAGENT-STOP>

帮助 PM 设计产品实验——从"我觉得这个方案更好"到"用实验验证假设"。

**核心原则**：实验不是证明你对，是发现真相。

## Iron Law（铁律）

| 铁律 | 违反后果 |
| --- | --- |
| 没有假设的实验不是实验 | 立即停止——补充 H0（原假设）和 H1（备择假设）后继续 |
| 没有成功标准的实验无法判断 | 立即停止——定义可量化的成功指标 + 阈值后继续 |
| 样本量不足的结论不可信 | 立即停止——估算最小样本量后继续（无法达到时标注"结果仅供参考"） |

## 反合理化

| 你可能在想的 | 真相 |
| --- | --- |
| "方案明显更好，不需要实验" | "明显"是偏见。实验验证"明显"的假设 |
| "没有足够用户做 A/B" | 样本量不足 ≠ 不能实验。可用定性实验、可用性测试替代 |
| "实验需要很长时间" | 快速实验（3-5 天）也能获得有价值的数据 |
| "结果会延迟上线" | 延迟一周 < 上线后方向错误返工一个月 |
| "只看转化率就行" | 单一指标误导。必须定义主要指标 + 护栏指标 |

## Entry Mode

### Guided（引导模式）— 8 步

1. **实验目标** — 验证什么假设？记录 H0 和 H1
2. **当前基线** — 当前指标表现？（无数据标注 [假设]）
3. **实验变量** — 改变什么？实验组 vs 对照组差异
4. **目标用户** — 实验对象是谁？占总用户比例？
5. **成功指标** — 主要指标 + 护栏指标 + 成功阈值
6. **样本量与时长** — 最小样本量估算 + 实验周期
7. **风险与降级** — 回滚方案
8. **分析计划** — 结果分析方式

### Quick（快速模式）— ≤ 2 个问题

**问题 1**: "你想验证什么假设？改动是什么？"
**问题 2**（可选）: "成功指标是什么？"

然后：自动生成 H0/H1 → 推断实验变量 → 建议指标+护栏 → 估算样本量 → 降级方案 → 标注 [假设]

## 执行流程

```
触发 pm-experiment
    ├── 1. 模式判断（Guided/Quick）
    ├── 2. 定义假设（H0 + H1）
    ├── 3. 实验设计（变量 + 人群 + 分流 + 时长）
    ├── 4. 指标定义（主要指标 OEC + 护栏指标 + 阈值）
    ├── 5. 样本量估算（基线 + 最小可检测效应）
    ├── 6. Iron Law 检查
    │     ├── H0/H1 存在？
    │     ├── 成功指标+阈值定义？
    │     └── 样本量估算？
    └── 7. 交付 + 建议下一步（→ pm-metrics 或 → pm-launch）
```

## Hard Bans（硬禁令）

以下模式在所有保真度级别均禁止出现。检测到时立即修正。

| # | 禁止模式 | 修正动作 |
| --- | --------- | -------- |
| HB-1 | 发布/方案无回滚方案 | 定义回滚触发条件+步骤+预计时间 |
| HB-2 | 指标体系无护栏指标（只优化不防副作用） | 补充 >=1 个护栏指标，定义底线和告警阈值 |
| HB-3 | 里程碑有精确日期但无缓冲 | 改为日期范围或预留 10-20% 缓冲 |
| HB-4 | 实验无原假设和备择假设 | 补充 H0（原假设）和 H1（备择假设） |
| HB-5 | P0 功能测试覆盖不完整 | P0 功能必须有正向+边界+异常测试覆盖 |


- [ ] H0 和 H1 已明确定义
- [ ] 当前基线数据已记录（或标注 [假设]）
- [ ] 实验变量（实验组 vs 对照组）明确
- [ ] 主要指标 + 护栏指标已定义
- [ ] 成功阈值可量化
- [ ] 最小样本量已估算
- [ ] 降级/回滚方案已列出
- [ ] 所有推断已标注 [假设]

## 实验类型选择

| 类型 | 适用场景 | 最低样本 |
| --- | --- | --- |
| **A/B 测试** | 测试清晰的 UI/功能变化 | ~1000/变体 |
| **落地页测试** | 构建前测试需求 | ~500 访问 |
| **原型测试** | 验证 UX、流程 | 5-8 人（定性） |
| **门房 MVP** | 手动服务测试价值主张 | 10-50 用户 |
| **假门测试** | 测量未开发功能的点击意愿 | ~500 访问 |
| **队列分析** | 分析现有功能对已用/未用组 | 按数据量 |

6 种实验类型详解见 `references/experiment-types.md`。
实验决策框架（通过/失败标准）见 `references/decision-framework.md`。

## 样本量经验法则

| 预期效应 | 每变体样本量 |
| --- | --- |
| 大效应（20%+） | ~1,000 用户 |
| 中等效应（10%） | ~3,000 用户 |
| 小效应（5%） | ~10,000+ 用户 |

## 常见错误

| 错误 | 后果 | 正确做法 |
| --- | --- | --- |
| 样本太小 | 无法检测有意义效应 | 用功效分析计算样本量 |
| 实验太短 | 未达统计显著性就停止 | 预先设定实验时长 |
| 偷看结果 | 反复检查导致假阳性 | 只在预设检查点分析 |
| 忽视护栏 | 负面副作用未发现 | 定义护栏指标并监控 |
| 太多变体 | 混淆结果 | 1 个控制组 + 1 个变体 |
