# 实验决策框架

> **统计显著性与实际显著性的决策规则**

---

## 决策矩阵

### 基础决策表

| 结果 | 统计显著性 | 实际显著性 | 决策 | 说明 |
|------|-----------|-----------|------|------|
| **Win** | ✅ p < 0.05 | ✅ 效应 ≥ 阈值 | **实施** | 变体明确更好，值得投入 |
| **Tie** | ✅ p < 0.05 | ❌ 效应 < 阈值 | **评估** | 有效果但太小，评估 ROI |
| **Loss** | ✅ p < 0.05 | N/A | **不实施** | 变体更差，回滚 |
| **Inconclusive** | ❌ p ≥ 0.05 | N/A | **不确定** | 样本不足或无差异，需要更多信息 |

---

## 统计显著性

### P值解释

**p值**: 观察到结果 (或更极端) 在零假设为真时的概率。

**零假设**: 两组无差异。

**示例**:
- p = 0.03 → 3% 概率看到这个差异如果实际上无差异
- p < 0.05 → 拒绝零假设，认为有显著差异

### 置信区间

**95% 置信区间**: 真实效应有 95% 概率落在范围内。

**示例**:
- 转化率提升: +2.5% [95% CI: 1.2%, 3.8%]

**解读**:
- 最小效应: +1.2%
- 最佳估计: +2.5%
- 最大效应: +3.8%

### 常见阈值

| 置信度 | p值阈值 | 应用场景 |
|--------|---------|----------|
| 90% | p < 0.10 | 探索性实验 |
| 95% | p < 0.05 | 标准实验 (推荐) |
| 99% | p < 0.01 | 高风险决策 |

## 实际显著性

### 定义

统计显著性 ≠ 实际重要性。

**示例**:
- 转化率从 10.00% 提升到 10.05%
- 可能统计显著 (大样本)
- 但实际意义微乎其微

### 效应大小

| 效应类型 | 相对提升 | 绝对提升 | 判断 |
|----------|----------|----------|------|
| **巨大** | > 20% | > 5pp | 非常重要 |
| **大** | 10-20% | 2-5pp | 重要 |
| **中** | 5-10% | 1-2pp | 有意义 |
| **小** | 1-5% | 0.2-1pp | 微小 |
| **极小** | < 1% | < 0.2pp | 可忽略 |

pp = percentage point (百分点)

### 最小可检测效应 (MDE)

**定义**: 你关心的最小效应。

**示例**:
- MDE = 2% 转化率提升
- 如果实验结果显示 1% 提升
- 即使统计显著，也不实际显著

## 决策场景详解

### 场景 1: Win (明确实施)

**特征**:
- p < 0.05
- 效应 ≥ MDE
- 置信区间不包括 0

**行动**:
- ✅ 全面发布
- ✅ 监控长期指标
- ✅ 规划后续优化

**示例**:
- 转化率: 10% → 12%
- p = 0.01
- 95% CI: [1.5%, 2.5%]
- MDE = 1%

---

### 场景 2: Tie (评估 ROI)

**特征**:
- p < 0.05 (统计显著)
- 效应 < MDE (实际不显著)

**评估因素**:
- 实施成本
- 维护成本
- 战略价值
- 风险

**决策矩阵**:

| 效应 | 成本 | 决策 |
|------|------|------|
| 小 | 低 | 考虑实施 |
| 小 | 高 | 不实施 |
| 中 | 低 | 实施 |
| 中 | 高 | 评估 |

**示例**:
- 转化率: 10% → 10.3%
- p = 0.03
- 效应: 0.3% (< MDE 1%)
- 实施: 需要3人月
- 决策: 成本高，效应小，不实施

---

### 场景 3: Loss (不实施)

**特征**:
- p < 0.05
- 变体更差
- 置信区间为负

**行动**:
- ❌ 回滚变体
- ❌ 保持 Control
- ✅ 分析为什么失败
- ✅ 记录学习

**示例**:
- 转化率: 10% → 9%
- p = 0.02
- 95% CI: [-1.5%, -0.5%]
- 决策: 回滚

---

### 场景 4: Inconclusive (不确定)

**特征**:
- p ≥ 0.05
- 置信区间跨越 0

**可能原因**:
1. 样本量不足
2. 效应太小
3. 变异太大
4. 实验时间太短

**行动**:

| 情况 | 行动 |
|------|------|
| 样本不足 | 延长实验 |
| 效应小 | 接受无差异，或测试更大变化 |
| 变异大 | 细分队列，减少噪音 |
| 时间短 | 延长到完整周期 |

**示例**:
- 转化率: 10% → 10.2%
- p = 0.30 (不显著)
- 95% CI: [-0.3%, +0.7%]
- 分析: CI 宽，样本可能不足
- 行动: 延长2周或接受结果

## 护栏指标

### 定义

确保实验不造成负面影响的指标。

### 常见护栏指标

| 类别 | 指标 | 阈值 | 超过则 |
|------|------|------|--------|
| **用户体验** | 跳出率 | +5% | 停止 |
| **性能** | 页面加载时间 | +200ms | 停止 |
| **业务** | 收入 | -2% | 停止 |
| **满意度** | NPS | -5点 | 停止 |
| **留存** | 次日留存 | -3% | 停止 |

### 监控计划

**频率**: 每日检查

**行动**:
- 设置告警
- 超过阈值立即停止
- 分析原因

---

## 不确定场景处理

### 混合结果

**定义**: 主要指标不显著，但次要指标显著。

**处理**:
1. 重新评估假设
2. 检查队列细分
3. 考虑延长实验
4. 依赖业务判断

### 外部因素

**定义**: 实验期间发生外部事件 (假期、营销活动、竞争)。

**处理**:
1. 识别外部因素
2. 分段分析 (排除影响期间)
3. 延长实验
4. 取消实验 (严重干扰)

### 技术问题

**定义**: 实验设置有 bug，数据有问题。

**处理**:
1. 停止实验
2. 修复问题
3. 重新开始或取消
4. 数据不可信，谨慎解读
