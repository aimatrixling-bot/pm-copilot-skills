# PM 求职深度指南

> JD 分析、面试准备和求职策略的实用参考。

---

## 1. JD 隐含期望解读

### 如何从 JD 字面读出隐含期望

| JD 字面 | 隐含期望 |
|---------|----------|
| "快节奏环境" | 加班可能较多，需要快速决策能力 |
| "从 0 到 1 经验" | 需要独立主导产品能力，不只是执行 |
| "数据驱动" | 必须有 A/B 测试或指标体系经验 |
| "跨团队协作" | 需要无权威影响力，能推动各方 |
| "创业心态" | 资源有限，需要自己动手 |
| "用户导向" | 必须有用户研究/访谈经验 |
| "0 到 1 阶段" | 产品可能不成熟，需要容忍不确定性 |
| "大厂背景优先" | 实际上可能更看重规范化流程经验 |

### JD 红旗（不建议投的信号）

| 红旗 | 含义 |
|------|------|
| "全栈 PM" | 可能什么都做，什么都不精 |
| 职责描述模糊 | 角色定位不清 |
| 要求 5+ 年但薪资对标初级 | 预算和期望不匹配 |
| 没有 PM 团队描述 | 可能是孤军奋战 |

## 2. 面试类型与准备

### 产品感面试

| 问题类型 | 示例 | 考察点 |
|----------|------|--------|
| 产品设计 | "如何改进电梯体验？" | 用户思维、结构化思考 |
| 产品策略 | "如何为老年人设计社交产品？" | 市场洞察、差异化 |
| 产品分析 | "DAU 下降了 20%，怎么排查？" | 数据思维、排查逻辑 |

### 行为面试

| 问题类型 | 示例 | 考察点 |
|----------|------|--------|
| 领导力 | "讲一个你推动团队决策的经历" | 影响力 |
| 冲突处理 | "如何处理与开发的分歧？" | 沟通协作 |
| 失败经验 | "讲一个失败的产品决策" | 反思能力 |

### 案例面试

| 结构 | 说明 |
|------|------|
| 澄清问题 | 确认理解正确（2 分钟） |
| 结构化分析 | 拆解为可分析的维度（5 分钟） |
| 深入分析 | 选择 2-3 个关键维度深入（10 分钟） |
| 综合结论 | 给出明确的建议（3 分钟） |
| 风险和权衡 | 诚实承认不确定性（2 分钟） |

## 3. PM 职级对照表

### 国内常见职级

| 职级 | 典型职责 | 年限参考 | 关键能力 |
|------|----------|----------|----------|
| PM 实习 | 协助调研、写文档 | 在校 | 学习能力 |
| 初级 PM | 执行功能模块 | 0-2 年 | 需求分析、PRD |
| 中级 PM | 独立负责产品线 | 2-5 年 | 数据分析、项目管理 |
| 高级 PM | 定义产品方向 | 5-8 年 | 战略思维、团队影响 |
| PM Director | 管理产品团队 | 8-12 年 | 组织设计、商业洞察 |
| VP Product | 产品战略决策 | 12+ 年 | 商业模式、行业洞察 |

### 面试准备清单

- [ ] 准备 3 个 STAR 故事（成功案例）
- [ ] 准备 1 个失败案例（含反思）
- [ ] 了解目标公司核心产品（用过最好）
- [ ] 准备 2-3 个问面试官的问题
- [ ] 模拟至少 1 次产品案例面试

## 4. 简历优化清单

| 检查项 | 通过标准 |
|--------|----------|
| 长度 | 1-2 页（不超过 2 页） |
| 量化 | 每条经历有 ≥ 1 个数字 |
| STAR | 每条成就用 STAR 结构 |
| 针对性 | 按 JD 调整重点（不是一份投所有） |
| 格式 | 清晰、无错别字、一致的排版 |
| 关键词 | 包含 JD 中的关键技能词 |
