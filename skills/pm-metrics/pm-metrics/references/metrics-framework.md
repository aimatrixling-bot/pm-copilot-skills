# 指标体系构建方法

> 从 North Star 到护栏指标的实用指南。

---

## 1. North Star Metric 选择

North Star Metric (NSM) 是衡量产品核心价值的单一指标。

### NSM 选择标准

| 标准 | 说明 |
|------|------|
| 反映核心价值 | 衡量用户获得的价值，不是公司获得的收入 |
| 可行动 | 团队能通过产品决策直接影响 |
| 领先指标 | 预测未来结果，不只是回顾过去 |
| 可理解 | 全公司都能理解 |

### 常见 NSM 示例

| 产品类型 | NSM | 为什么 |
|---------|-----|--------|
| 社交 | 日活跃用户 | 反映用户粘性 |
| SaaS | 周活跃团队数 | 反映产品嵌入深度 |
| 电商 | 月购买用户数 | 反映转化和复购 |
| 内容 | 用户消费内容时长 | 反映内容价值 |
| 工具 | 周完成任务数 | 反映工具效用 |

### 常见错误 NSM

| 错误 NSM | 问题 |
|----------|------|
| 注册用户数 | 包含大量不活跃用户 |
| 页面浏览量 | 虚荣指标，不反映价值 |
| 收入 | 滞后指标，不可行动 |

## 2. 指标分层体系

```
North Star Metric（北极星指标）
    ├── 输入指标（驱动 NSM 的上游）
    │   ├── 新用户注册转化率
    │   ├── 功能发现率
    │   └── 首次使用成功率
    ├── 输出指标（衡量 NSM 的下游）
    │   ├── 用户留存率
    │   ├── NPS 评分
    │   └── 付费转化率
    └── 护栏指标（防止副作用的底线）
        ├── 崩溃率
        ├── 客诉率
        └── 加载时间
```

### 指标数量建议

| 层级 | 数量 | 说明 |
|------|------|------|
| NSM | 1 个 | 只有一个核心 |
| 输入指标 | 3-5 个 | 可直接影响的 |
| 输出指标 | 2-3 个 | 衡量结果的 |
| 护栏指标 | 2-3 个 | 不能跌破的 |

---

## 3. 指标定义模板

每个指标必须包含：

| 维度 | 说明 | 示例 |
|------|------|------|
| 名称 | 清晰可理解 | "7日留存率" |
| 定义 | 精确计算公式 | 注册后第7天仍登录的用户 / 注册用户 |
| 数据来源 | 从哪取数据 | 用户行为日志表 |
| 计算方式 | SQL/公式 | COUNT(DISTINCT user_id) WHERE ... |
| 当前值 | 最新基线 | 35% |
| 目标值 | 预期达到 | 40% |
| 更新频率 | 多久算一次 | 每日 |
| 负责人 | 谁关注这个指标 | 增长团队 |

---

## 4. 目标设定方法

### OKR 方式

```
Objective: 提升用户参与度
  KR1: DAU 从 10万提升到 15万
  KR2: 人均使用时长从 5分钟提升到 8分钟
  KR3: 7日留存从 35% 提升到 40%
```

### SMART 检查

| 维度 | 检查 |
|------|------|
| Specific | 目标是否具体？ |
| Measurable | 能否量化？ |
| Achievable | 是否可达？基于什么假设 |
| Relevant | 是否与战略对齐？ |
| Time-bound | 什么时候达到？ |

---

## 5. 常见错误

| 错误 | 后果 | 正确做法 |
|------|------|----------|
| 追踪太多指标 | 信息过载，无法聚焦 | NSM + 3-5 输入 + 2-3 护栏 |
| 虚荣指标 | 看起来好但不指导行动 | 只追踪可行动指标 |
| 无基线 | 不知道当前在哪 | 先建立基线再设目标 |
| 无护栏 | 优化 A 却损害 B | 每个核心优化都设护栏 |
| 不更新 | 指标过时失去意义 | 每季度复查指标体系 |
